java static关键字

程序设计 java JVM虚拟机 2018/11/08

在Java关键字中，static是我们非常熟悉的一个关键字，在一定的环境下使用，可以提高程序的性能、优化程序的结构。static表示“全局”或者“静态”的意思，主要用来修饰成员变量、成员方法和静态块。

一、修饰成员变量

static修饰的变量称为静态变量或类变量，被static修饰的变量独立于该类的任何对象，它不依赖类特定的实例，被类的所有实例所共享。在使用时直接用类名.变量名的方式访问该类的静态成员，静态变量在类装载的时候JVM就会为其分配内存空间，并且一个static变量只会有一个内存空间，也就是说无论有多少个类的实例，这些实例中的这个static变量会共享同一个内存空间。

1.静态成员变量在内存中的分布

由上图可以看出main方法中user1和user2两个变量引用的对象分别存储在堆的不同区域，所以他们之间不会互相干扰。对象的成员变量由每个对象自己保存，而对象的方法是指向了静态方法区的方法定义。

方法区是Java堆中的一个逻辑区域，也是所有线程所共享的区域，它用于存储已被虚拟机加载的类信息、常量、静态变量、即时编译器编译后的代码等数据。

同样的静态变量count也是User类的所属，而不属于某个具体对象，也就是说如果其中一个对象修改了count属性，其它的对象都会受到影响。

2.static变量的加载顺序

上述代码中，User类中创建了四个Student变量，两个普通成员变量，两个static成员变量，当我们在创建User对象的时候（执行main方法），static修饰的成员变量首先被初始化，随后是普通成员，最后调用User类的构造方法完成初始化。也就是说，在创建对象时，static修饰的成员会首先被初始化，而且在有多个static修饰的成员时，JVM会按照他们出现的先后顺序进行初始化。

二、修饰成员方法

相对于修饰成员属性，修饰成员方法在数据的存储上面并没有多大的变化，因为我们从上图可以看出，方法本来就是存放在类的定义中。static修饰成员方法最大的作用就是可以使用类名.方法名的方式来操作方法，避免了先要new出对象的繁琐和资源消耗，因此静态方法中不能用this和super关键字，不能直接访问类的实例变量和实例方法（非static的成员变量和成员方法）。

static方法的初始化顺序

在上面的代码中，在执行main方法时，我们在没有创建对象，只是通过类去调用类的静态方法时，尽管该方法没有使用到任何的类成员，类的静态成员还是在方法调用之前就被初始化了，这说明，我们第一次使用一个类的时候，就会触发该类的静态成员初始化。而当我们new该类的对象时，普通的成员变量被初始化，而static修饰的成员不会再被初始化，也就是说static修饰的类成员，在类被加载的时候就会被初始化，并且只会被初始化一次，不会被多次初始化。

三、static代码块

static代码块也叫静态代码块，是类中独立于成员的static语句块，可以有多个，JVM加载类的时候会按照它们在类中出现的先后顺序一次执行它们。每个代码块只会被执行一次。

内容补充:

在JDK1.8之前，方法区由永久代实现，主要存放类的信息、常量池、方法数据、方法代码等。在JDK1.8之后，取消了永久代，提出了元空间，并且常量池、静态成员变量等迁移到了堆中；元空间不在虚拟机内存中，而是放在了本地内存中。
